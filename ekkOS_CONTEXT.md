# apps/docs

This directory contains the official, canonical documentation for **ekkOS**, the persistent memory layer for AI coding assistants. It serves as an operator manual for users, providing guides, API references, and integration instructions for platforms like Claude Code, Cursor, and VS Code.

## Architecture
- **Runtime**: Node.js (Next.js 16.0.10, React 19.1.0)
- **Entry points**: `app/layout.tsx` (root layout), `app/page.tsx` (homepage redirecting to `/introduction`)
- **Key dependencies**: 
  - `next`: Framework for the documentation site.
  - `lucide-react`: Icon library for UI elements.
  - `clsx` / `tailwind-merge`: Utility for dynamic class management.
- **Build system**: Yarn workspaces (invoked from monorepo root via `yarn workspace @ekkos/docs build`).
- **Deploy target**: Vercel (deployed to `docs.ekkos.dev`).

## Key Files
- `app/layout.tsx`: Root layout defining the ekkOS branding and global styles.
- `app/page.tsx`: Entry point that redirects users to the `/introduction` route.
- `next.config.js`: Next.js configuration, including path redirects and build-time ignore settings.
- `tailwind.config.ts`: Defines the ekkOS design system (void background, specific accent colors).
- `content/docs/`: Directory containing the core MDX documentation files.
- `app/components/DocsLayout.tsx`: Shared layout wrapper for documentation pages.
- `app/ekkOS_CONTEXT.md`: Context file for AI agents operating within this directory.
- `DEPLOYMENT.md`: Instructions for deploying the documentation site.

## API / Exports
The system is a static/server-rendered documentation site. It exposes the following primary routes:
- `/introduction`: High-level overview of ekkOS.
- `/quickstart`: Getting started guide.
- `/api-reference`: Documentation for MCP tools and webhooks.
- `/concepts`: Deep dives into memory layers, directives, and the "golden loop."
- `/integrations`: Setup guides for Claude Code, Cursor, VS Code, and Windsurf.
- `/cli-reference`: Documentation for the ekkOS CLI.

## Environment Variables
- `.env.local`: Used for local development configuration (standard Next.js pattern).

## Conventions
- **Documentation Philosophy**: Focus on "what users can do" and "what users will observe." Explicitly avoid documenting internal implementation details, scoring algorithms, or learning mechanisms.
- **File Organization**: 
  - `app/` contains the Next.js routing structure.
  - `content/docs/` contains raw MDX content.
  - `components/` contains reusable UI components (e.g., `AuroraBackground`, `CopyBlock`).
- **Styling**: Uses a "cyberpunk" aesthetic with a `void` (#050508) background and specific purple/blue/pink accents.
- **Build Process**: Uses `yarn workspace @ekkos/docs build` from the monorepo root.

- **Build Errors**: `next.config.js` is configured to `ignoreBuildErrors: true` and `ignoreDuringBuilds: true` for ESLint. This may mask type or linting issues during deployment.
- **Image Optimization**: `images.unoptimized` is set to `true`, which is necessary for certain Vercel deployments but may impact performance if not managed.
- **Documentation Boundary**: Do not attempt to document internal system architecture or learning pipelines; the documentation is strictly an operator manual.
- **Redirects**: The root `/` is redirected to `/introduction`. Ensure new top-level pages are accounted for in `next.config.js` if necessary.
