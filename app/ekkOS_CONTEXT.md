# apps/docs/app

This directory contains the documentation application for the ekkOS platform. It serves as the primary source of truth for developers and users, providing conceptual guides, API references, integration tutorials, and architectural documentation for the ekkOS ecosystem. It is built as a Next.js application that mirrors the platform's "cyberpunk" design language.

## Architecture
- **Runtime**: Node.js (Next.js 15+ App Router)
- **Entry points**: 
    - `layout.tsx`: Root layout and global design system provider.
    - `page.tsx`: Documentation landing page.
- **Key dependencies**: 
    - `next`: Framework for routing and server-side rendering.
    - `react`: UI component library.
    - `framer-motion`: Motion primitives for the cyberpunk UI aesthetic.
    - `lucide-react`: Iconography used throughout the documentation.
    - `clsx` / `tailwind-merge`: Utility management for the design system.
- **Build system**: Turborepo (monorepo integration) with Vercel deployment.
- **Deploy target**: Vercel (Production environment for `docs.ekkos.io`).

## Key Files
- `layout.tsx`: Defines the global structure, including navigation and the "cyberpunk" design system wrapper.
- `page.tsx`: The documentation landing page.
- `components/DocsLayout.tsx`: Shared layout wrapper for documentation pages, handling sidebar and content area.
- `components/CopyBlock.tsx`: Reusable component for code snippets with copy functionality.
- `concepts/golden-loop/page.tsx`: Documentation for the core ekkOS "Golden Loop" architectural pattern.
- `api-reference/mcp-tools/page.tsx`: Documentation for Model Context Protocol (MCP) tool definitions.
- `integrations/claude-code/page.tsx`: Integration guide for Claude Code.
- `ekkOS_CONTEXT.md`: The primary context file for AI agents to understand the codebase.

## API / Exports
The application exposes documentation routes mapped to the file system:
- `/introduction`: High-level overview of the ekkOS platform.
- `/concepts/*`: Deep dives into memory layers, privacy, and the golden loop.
- `/api-reference/*`: Technical documentation for MCP tools, webhooks, and CLI.
- `/integrations/*`: Guides for connecting ekkOS with Cursor, Windsurf, VSCode, and Claude.
- `/quickstart`: Onboarding path for new users.

## Environment Variables
*Note: As a documentation app, it relies primarily on build-time environment variables.*
- `NEXT_PUBLIC_SITE_URL`: Base URL for canonical links.
- `NEXT_PUBLIC_ANALYTICS_ID`: Tracking identifier for documentation usage.

## Conventions
- **File-based Routing**: Uses Next.js App Router; every `page.tsx` represents a route.
- **Design System**: Follows a "cyberpunk" aesthetic; components are styled to match the broader ekkOS marketing design system.
- **Documentation Structure**: Organized by intent (Concepts, Guides, API Reference, Integrations).
- **Componentization**: Shared UI elements (Copy buttons, layout wrappers) are located in `components/`.
- **Living Docs**: The repository utilizes `ekkOS_CONTEXT.md` files to maintain synchronization between code and documentation.

## Gotchas
- **Sync Latency**: Documentation is manually synced with platform features; ensure that `packages/ekkos-core` changes are reflected in the `concepts/` directory.
- **Design System Fragility**: The "cyberpunk" UI overhaul is highly specific; custom CSS in `globals.css` may conflict with standard Tailwind utility classes if not handled carefully.
- **Tool Count**: Documentation references a specific tool count (currently 32); ensure any new CLI tools added to `packages/ekkos-cli` are updated in the docs to maintain accuracy.

## Connected Systems
- **`packages/ekkos-cli`**: Documentation for CLI commands is derived from this package.
- **`packages/ekkos-core`**: Architectural concepts (Memory Layers, Golden Loop) are defined here.
- **`mcp-servers-*`**: API references for MCP tools are generated based on these server implementations.
- **`apps-marketing-app`**: Shares the design system and branding assets.
- **`apps-memory-api` / `apps-memory-lib`**: High co-change coupling; updates to memory architecture require corresponding documentation updates in this directory.
